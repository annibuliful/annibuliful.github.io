Functional Reactive Programming ยิ่งใหญ่ไหลเรื่อยๆ ตอนที่ 1

สวัสดีครับทุกคนหลังจากที่ได้มีโอกาสลองเล่น RxJS แล้วรู้สึกนี้มันเป็นอะไรที่ยิ่งใหญ่มากๆ แต่ด้วยความที่เป็นตัวผม ผมเลยรู้สึกว่าอยากรู้มากขึ้นกว่านั้นอีกเพราะผมเข้าใจว่ากว่าที่มันจะเป็น RxJS ได้มันต้องมีอะไรที่ยากกว่านั้นอยู่เบื้องหลังแน่นอนผมเลยพยายามหาอ่านตาม paper และ articles ต่างๆมาพอสมควรจนคิดว่าตัวเองเข้าใจความเป็น FRP มาแล้วระดับนึง

ก่อนอื่นต้องขอบอกก่อนว่าตัวมันเองเกิกจาก paper ของคน 2 คนคือ Conal Elliott[1] และ Paul Hudak[2] ซึ่งเป็น computer scientist ที่ทำในหัวข้อเรื่อง Functional Reactive Animation เรามาเริ่มต้นคำจำกัดความกันก่อนเลย

What is Functional Reactive Programming?

Behaviors => Behaviors are time-varying, reactive values

Events => events are sets of arbitrarily complex conditions, carrying possibly rich information

ใน paper ได้เขียนถึง 2 สิ่งหลักๆด้วยกันคือ Behaviors กับ Events โดยเริ่มที่ Behavior(พฤติกรรม) หมายถึงว่าจะเปลี่ยนแปลงค่าตามกาลเวลาส่วน Events(เหตุการณ์) หมายถึงเหตุการณ์ที่ทำให้ค่านั้นๆเปลี่ยนไปยกตัวอย่างเช่น เรากำลังเล่นเกมต่อสู้อยู่แล้วเราถูกโจมตีทำให้หัวใจของเราลดไป 2 ดวงนั้นหมายความว่า Event คือการถูกโจมตีและ Behavior คือทำให้หัวใจเราลดไปเป็นจำนวน 2 หน่วยนั้นเอง

จากรูปทุกคนจะเห็น 3 อย่างคือ Event, State, Time โดยจะเห็นว่าเมื่อใดก็ตามที่เกิด Event ขึ้นมานั้นจะมีข้อมูลใหม่ๆเข้ามาซึ่งนั้นมันอาจจะทำให้ค่าเดิมของ state มีค่าที่เปลี่ยนแปลงไปตามเวลาที่เกิด Event นั้นๆขึ้นก็ได้ ยกตัวอย่างเช่น เรากดแป้นพิมพ์ A แล้วเราจะเห็นตัวอักษร A ขึ้นที่หน้าทันที

ด้วยความที่มันเป็น Function Reactive Programming นั้นหมายความว่าเราสามารถใช้ map filter reduce ในการเปลี่ยนแปลงของข้อมูลที่เกิดขึ้นได้
